### **项目概述**
本项目是一个基于 Python 的 GUI 工具，主要用于从智能电表的需求解析 Excel 文档中提取出厂参数、显示项等数据，并与标准数据模型进行匹配和比对。它集成了 DLMS/COSEM 协议通讯能力，可以直接读取电表数据并与文档要求进行自动校对。

---

### **项目结构**
项目采用了清晰的分层架构，将界面、逻辑、协议处理和资源文件分离。

- **根目录**
  - [main.py](file:///e:/code/需求文档出厂参数项提取脚本/main.py): 程序入口，负责初始化环境和启动 GUI。
  - [updates_notes.txt](file:///e:/code/需求文档出厂参数项提取脚本/updates_notes.txt): 版本记录与功能更新日志。
  - [requirements.txt](file:///e:/code/需求文档出厂参数项提取脚本/requirements.txt): 依赖包列表。
  - [.trae/rules/project_rules.md](file:///e:/code/需求文档出厂参数项提取脚本/.trae/rules/project_rules.md): Trae IDE 的项目特定 AI 规则。

- **核心目录**
  - **`src/`**: 源代码主目录。
    - **`ui/`**: 存放 Tkinter 界面代码，如 [main_window.py](file:///e:/code/需求文档出厂参数项提取脚本/src/ui/main_window.py)。
    - **`logic/`**: 业务逻辑层。包含 Excel 处理 ([excel_processor.py](file:///e:/code/需求文档出厂参数项提取脚本/src/logic/excel_processor.py))、OBIS 匹配 ([obis_matcher.py](file:///e:/code/需求文档出厂参数项提取脚本/src/logic/obis_matcher.py)) 和数据比对逻辑。
    - **`core/`**: 底层通讯层，如 [dlms_client.py](file:///e:/code/需求文档出厂参数项提取脚本/src/core/dlms_client.py)。
    - **`mappings/`**: 静态数据映射。存放 OBIS 码表、单位转换、事件码定义等。
    - **`pypack/`**: 自定义的协议处理库，包含 DLMS 协议的编解码逻辑。

- **辅助目录**
  - **`assets/`**: 资源文件。存放图标、常用 OBIS 匹配表、单位定义等辅助 Excel/文本文件。
  - **`scripts/`**: 独立脚本。用于执行特定的自动化任务，如 [generate_formatted_columns.py](file:///e:/code/需求文档出厂参数项提取脚本/scripts/generate_formatted_columns.py)。
  - **`build_system/`**: 打包系统。包含 PyInstaller 的 `.spec` 模板和自动化打包脚本。
  - **`docs/` & `reference_docs/`**: 输入文档示例和 DLMS/COSEM 技术参考文档。

### **核心开发规则**
在本项目中，必须遵守以下开发规范：

- **版本与更新管理**: 
  - 每次代码变更后，必须在 [updates_notes.txt](file:///e:/code/需求文档出厂参数项提取脚本/updates_notes.txt) 中记录更新内容。
  - 新功能或 Bug 修复需按约定增加版本号（如 V2.95 -> V2.96）。
- **语言规范**:
  - **注释**: 强制要求使用**中文代码注释**。
  - **界面**: 所有 GUI 交互、提示信息、按钮文本必须使用**中文**。
  - **AI 交互**: 与 AI 助手沟通时使用中文。
- **编码实践**:
  - **模块化**: 优先搜索 `src/logic` 或 `src/mappings` 中已有的函数或接口，避免重复造轮子。
  - **健壮性**: 在处理 Excel 写入时，采用“自动寻找空白列追加”的逻辑，而非固定列号，以增强脚本的兼容性。
  - **Windows 适配**: GUI 应用需调用 `ctypes` 设置 `AppUserModelID`，以确保任务栏图标正常显示。

### **技术栈参考**
- **编程语言**: Python 3.x
- **GUI 框架**: Tkinter (内置，轻量级，无需额外依赖)
- **数据处理**: 
  - `openpyxl`: 用于读写 `.xlsx` 格式的 Excel 文件。
  - `re`: 大量使用正则表达式处理 OBIS 码和文本清洗。
- **底层通讯**: 
  - `pyserial`: 用于串口通讯。
  - `pycryptodome`: 提供 DLMS 安全认证所需的加密算法。
- **打包工具**: `pyinstaller`

### **典型代码模式 (Pattern)**
- **动态版本提取**: 程序启动时从 `updates_notes.txt` 自动读取最新版本号。
- **异步处理**: 在 GUI 中执行耗时的比对或通讯操作时，使用 `threading` 配合 `Queue` 或回调函数，防止界面卡死。
- **路径自适应**: 在 [main.py](file:///e:/code/需求文档出厂参数项提取脚本/main.py) 中动态将 `src` 目录加入 `sys.path`，确保在不同环境下运行时的导入正确性。
